Problem & Goal

Manual voter verification causes long queues, human error, and security weaknesses. This prototype demonstrates a safer, faster verification flow combining QR/NFC, ID lookup, and biometric-assisted confirmation with strong auditing and privacy-preserving practices.

Design highlights & security considerations

  • Minimal PII exposure: Terminals only fetch the minimum fields needed; photos and sensitive attributes stay encrypted and are never exposed to third-parties.
  • One-way proofs: Use of hashed commitments (e.g., Bloom filters / salted hashes) for fast local matching without revealing raw identifiers.
  • Tamper-evidence: Each verification event is hashed and signed by a local HSM or secure element; chain of custody is logged.
  • Privacy by design: Biometric checks are optional and used only with explicit consent; audit trails contain event IDs and pseudonyms, not raw biometric templates.
  • Fallbacks: Offline operation with reconciliation, manual override paths with countersigned receipts for exceptional cases.

Prototype limitations

This page is a frontend demo — a real system requires secure server endpoints, properly provisioned keys, compliance checks, accessibility testing, and rigorous security audits before deployment.
